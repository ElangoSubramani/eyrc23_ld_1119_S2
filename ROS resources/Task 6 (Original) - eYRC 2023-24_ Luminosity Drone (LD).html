Task 6

Original Configuration

Aim

The objective of this task is to identify organisms present on the surface of the exoplanent and report back to base with swift drone.

Prerequisites

  • It is mandatory to have completed task 5 successfully.

  • It is assumed that you haven't moved the arena or the overhead camera after the mapping in Task 5. If you have moved the arena, mapping must be done again. Please don't move the arena once it is mapped.

Installations

  • Update the luminosity_drone repo from github for ROS2.

    cd ~/colcon_ws/src/luminosity_drone
git pull origin ros2_pkg

  • Download the original configuration zip file from the portal. Unzip the file, copy the executable named spawner into the location colcon_ws/src/luminosity_drone/alien_finder/src/.

  • Build the workspace

    cd ~/colcon_ws
colcon build
source install/setup.bash

    üìå Note :
    Follow the steps for mapping the arena, only if you have moved the arena. Otherwise no need to repeat again.

  • If you have moved the arena after task 5, then in a new terminal, run the following commands to map the arena:

    • In one terminal, run:
    ros1
ros1_ws
roslaunch luminosity_drone task_4a.launch
    • In another terminal, run:
    ros1
ros1_ws
rosrun luminosity_drone arena_mapper.py

    This creates a .json file that has the WhyCon coordinates mapped to the arena coordinates

    üìå Note :
    After this step, neither the arena nor the overhead camera should be moved (use tape to stick arena to ground). If moved, record the co-ordinates again by running the above script.

Problem Statement

In this task you will do a full implementation of the theme following the rules given in the rulebook.

  • The organisms are simulated using LED modules in the arena(exoplanet surface).

  • Identify the location (whycon coordinate) of the organism on the surface of the exoplanet.

  • Publish the location and the type of organisms onto a ROS2 topic.

  • Indicate the type of organism found using onboard LEDs and buzzer.

Procedure

  • The drone should land at the position shown in the figure given below (i.e. bottom right corner of arena).


  • Connect the usb camera to laptop/PC. Launch the usb camera, WhyCon node using the launch file.

  • Run task_4a.launch in ROS1 in a terminal followed by ros1_bridge node in another terminal.

    ros1
ros1_ws
roslaunch luminosity_drone task_4a.launch

    ros1
rosbridge
ros2 run ros1_bridge parameter_bridge

  • In a new terminal, run the following command to generate the configuration for the placement of the LED modules on the arena

    ros2 run alien_finder spawner

    spawner generates different configuration for the placement of the LED modules everytime you run it and the takeoff point as shown in the figure below (One of the four takeoff points will be provided). This will generate two different configurations for placing two different LED modules.



  • Keep all the 4 types of aliens (LED modules) ready, as soon as you run the above node, keep the respective alien type at the respective coordinate. For eg, the configuration generated by the spawner is alien_b at C4, then take the alien_c (4 LED module), turn it on, and place it at the cooridnate C4 and leave the arena. Once placed, you can then run the Task_6.py that flies the drone.

  • Edit the blank python script Task_6.py in the colcon_ws/src/luminosity_drone/alien_finder/scripts/ folder and complete the script to fly the drone through the arena and detect the organisms.

  • Write a search algorithm for the Swift drone to scan the entire arena to find and identify the different types of organisms.

  • Throughout the entire flight, deploy image processing with the onboard camera on the Swift drone in an effort to detect and identify organisms.

  • As soon as you detect the organism in the camera frame, try to align the centroid of the LED to the center of the camera frame.

  • The different organism type according to the no. of LEDs are given below:

    No. of LEDs Organism Type
    2 alien_a
    3 alien_b
    4 alien_c
    5 alien_d

  • Once you are confident about the alingment, you have to publish the type of organisms and the Whycon coordinates onto the the following ROS2 topic /astrobiolocation having message type loc_msg/msg/Biolocation.

    loc_msg/msg/Biolocation Message type:

    arun@eyantra:~/colcon_ws$ ros2 interface show loc_msg/msg/Biolocation 
  string organism_type
  float64 whycon_x 
  float64 whycon_y 
  float64 whycon_z

  • During the same time, you have to indicate the type of organism found using on-board LED and buzzer.

  • Each organism can be represented using LED blinks and buzzer beeps as follows:

Organism TypeLED BlinksBuzzer Beeps
alien_a22
alien_b33
alien_c44
alien_d55

  • Each LED blink should have a gap of 0.2s between them in case of multiple blinks.

  • Each buzzer beep should have a gap of 0.2s between them in case of multiple beeps.

    Refer rulebook for the same.

  • You have to do publishing onto the ROS2 topic and indication using on-board LED and buzzer for all 2 organisms detected.

  • After your script is complete, in a new terminal, run the Task_6.py:

    ros2 run alien_finder Task_6.py

  • If you want to use Plotjuggler and Pid_tune_gui for tuning purposes, you can run the following commands:

    ros2 run plotjuggler plotjuggler
ros2 run pid_tune pid_tune_drone.py

üìå Note

  • Each time an organism is located, publish onto the ROS2 topic called /astrobiolocation only once.
  • The time taken to identify the organism is NOT taken as a parameter for scoring.
  • The total evaluation time is 120 secs, so be careful to complete the task within that time frame.
  • For successful completion of the task, you have to run spawner before you run Task_6.py so that your code is able to detect an organism anywhere in the arena.

Warning:
Participants are advised not to hardcode or copy from others. If found out, your team will be disqualified.

To make sure hard coding is not possible, the spawner makes sure a random configuration with random aliens and random locations is generated. To have a valid submission for this task, the configuration given by launch file has to match aliens placed on the arena, else the evaluator will not give a valid score.

In final task, submissions won't be taken into consideration if rope is attached to the drone.

Recording and Submission Instructions

Recording Instructions

  • Step 1: Once your are confident that your script is ready then, use the launch file hat runs the configuration generation node as well as the rosbag recording.

    ros2 launch alien_finder task_6_submission.launch.py

    This will generate the configuration for the placement of the LED modules on the arena and start the rosbag recording. You have to place the LED modules on the arena within 30s. As soon as you have placed the LED modules, run your script in a new terminal:

    ros2 run alien_finder Task_6.py

    üìå The rosbag files will be saved in the directory from which you launch the command, so find the .db3 and .yaml files in the directory the terminal is opened from.

  • Step 2: Record the screen of pc/laptop before launching the above launch file. Split the windows on the screen such that half of the screen should have terminals and other half should have the camera feed from the ceiling. Record the video until the drone lands.

    üìå The video should show the configuration generated by the launch file on screen and the person placing the alien on the arena using the overhead camera.

  • Step 3: Record another video that captures the led module and the buzzer beeping. Place the recorder such that it is behind the drone and the led module is visible at all times. Now merge both videos and submit it on the portal.
