Task 3b

ROS2 Foxy Installation & Camera Calibration

Aim:

Install ROS2 Foxy and interface USB camera that is provided with hardware kit with ROS2 and calibrate it.

ROS2 Foxy Installation

  • Make sure you have a locale which supports UTF-8. If you are in a minimal environment (such as a docker container), the locale may be something minimal like POSIX. We test with the following settings. However, it should be fine if you’re using a different UTF-8 supported locale.

    locale  # check for UTF-8

sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

locale  # verify settings

Setup Sources

  • You will need to add the ROS 2 apt repository to your system.

  • First ensure that the Ubuntu Universe repository is enabled.

    sudo apt install software-properties-common
sudo add-apt-repository universe

  • Now add the ROS 2 GPG key with apt.

    sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

  • Then add the repository to your sources list.

    echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

Setup Sources

  • Update your apt repository caches after setting up the repositories.

    sudo apt update

  • ROS 2 packages are built on frequently updated Ubuntu systems. It is always recommended that you ensure your system is up to date before installing new packages.

    sudo apt upgrade

  • Desktop Install : ROS2, RViz, demos, tutorials.

    sudo apt install ros-foxy-desktop python3-argcomplete

  • Development tools: Compilers and other tools to build ROS packages

    sudo apt install ros-dev-tools

  • The next step is very crucial, where you need to disable ROS1 and allow ROS2 to be used by the programs. Open ~/.bashrc located in the HOME directory in any editor of your choice. Navigate to the line where you have sourced the two files for ROS1, comment them and add new lines for ROS2

    source /opt/ros/foxy/setup.bash

TIPS: Try running roscore on the terminal,rosmaster starts if you are using noetic and if there is an error that implies you have switched to Foxy.Also you can check for Foxy by running ros2 topic list

  • Source bashrc with the following command or start a new terminal:

     source ~/.bashrc

  • Create a colcon workspace, the ROS2 eqivalent of catkin workspace in ROS1 

    cd 
mkdir -p colcon_ws/src 
cd colcon_ws
colcon build

    This will create a build and install directory similar to build and devel generated by catkin build in ROS1

  • Next step is to clone the luminosity_drone ROS2 package in the workspace

    cd ~/colcon_ws/src
git clone -b ros2_pkg https://github.com/eYantra-Robotics-Competition/luminosity_drone 
cd ~/colcon_ws
colcon build

  • Now add the following line in .bashrc

    source ~/colcon_ws/install/setup.bash

  • Source bashrc with the following command or start a new terminal:

     source ~/.bashrc

Calibration Procedure :

Testing usb camera

  • Connect the USB wire to camera and other end to laptop/pc. To test the camera, you need a image viewer in linux. One of the camera viewer is guvcview. To install, follow this command on terminal.

    sudo apt install guvcview

  • To open guvcview, type guvcview on terminal. Go to video section and select USB camera. This will test the camera.

Interfacing usb camera with ROS2

  • Next step is to interface the camera with ROS2 as we will be fetching the image and processing via rosnodes. To do this, we will be using a ros package named usb_cam. Usually this package comes pre installed with ros-foxy-desktop-full. Just to be sure, you can re install the package by typing

    sudo apt install ros-foxy-usb-cam

  • Next step is to identify the camera device. Type

    ls /dev/video*

    This will list the video devices attached to your system, to identify the usb camera, disconnect the camera and run the command, and then reconnect the camera and run the same command to check which device is getting added after connecting the camera. For eg. the usb camera is video2. Update ros package of luminosity_drone from github and find a new launch file named demo_launch.py

  • Go to config folder of luminosity_drone and edit the params.yaml file and in place of video1 put video2.After editing the video device, launch the demo_launch.py file

    ros2 launch luminosity_drone demo_launch.py

    This will start a window of the usb camera and publish the topics with camera image frames . You can now use the topic /image_raw.

Attaching camera to ceiling

  • Attach the camera to the ceiling using double side tape. Make sure the height of ceiling from the floor is minimum 10 ft. Little more than 10 ft is better, you can go till 12 ft.

Camera Calibration Process

  • You will need a checkerboard in order to calibrate your camera. Print out the image on an A4 sheet given on this link

  • You must install the camera calibration package in ROS2. Open a terminal and type the following command (Type the commands):

    sudo apt install ros-foxy-camera-calibration

  • Run the following two commands on separate terminals:

    ros2 launch luminosity_drone demo_launch.py
ros2 topic list

You should see ‘/image_raw’ and ‘/camera_info’.

  • Next, run the following command:

    ros2 run camera_calibration cameracalibrator --size 6x8 --square 0.025 \
  --ros-args -r image:=/image_raw

  • You should now see a new window as shown in Figure 3.

  • Hold up the checkerboard in front of the camera. A zig-zag line should be displayed on the checkerboard. You must now perform the following calibrations by the completing the given steps: a. X axis – Move the checkerboard left to right and right to left. b. Y axis – Move the checkerboard top to bottom and bottom to top. c. Size – Move the checkerboard close to away and away to close from the camera. d. Skew – Tilt the checkerboard in all directions.

    Note: The more sample you take, the better is the output.

    The following figures elaborate on this:

  • You must perform all these steps until you get maximum green for X, Y, Size and Skew in both directions on the panel on the right-hand side. When complete, your final progress should look like Figure 8.

  • When you get green progress for X, Y, Size and Skew, your ‘CALIBRATE’ button will be highlighted. Click that button in order to generate the calibration matrix. This might take some time so please wait while it generates the matrix. It might appear your computer has hung, but that is not the case.

  • Once the calibration matrix is generated, the ‘SAVE’ and ‘COMMIT’ button are highlighted. Hit ‘SAVE’ and then ‘COMMIT’. This saves your matrix.You can also see the result in terminal.

  • Data is saved to /tmp/calibrationdata.tar.gz.To use the the calibration file unzip the calibration.tar.gz

  • In the folder, images used for calibration are available and also “ost.yaml” and “ost.txt”. You can use the yaml file which contains the calibration parameters as directed by the camera driver.

  • Copy the contents from ost.yaml to camera_info.yaml file located in the config folder of luminosity_drone.
